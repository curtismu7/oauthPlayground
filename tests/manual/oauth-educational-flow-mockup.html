What is the Authorization Code Flow?

The Authorization Code Flow is the most secure OAuth 2.0 flow for web applications. It's designed for applications that can securely store a client secret and can make server-to-server requests.

Key Benefits:

  • Most secure OAuth flow
  • Client secret never exposed to user agent
  • Supports PKCE for additional security
  • Can obtain refresh tokens for long-lived access

Pro Tip: Understanding Scopes

Scopes define what permissions your application is requesting. In OpenID Connect, openid is always required. Common scopes include:

  • openid - Required for OpenID Connect
  • profile - Access to basic profile information
  • email - Access to email address
  • offline_access - Ability to get refresh tokens

What is PKCE?

PKCE (Proof Key for Code Exchange) is a security extension to OAuth 2.0 that provides additional protection for public clients and mobile applications. It prevents authorization code interception attacks.

PKCE Flow:

  1. Client generates a random code_verifier
  2. Client creates a code_challenge from the verifier
  3. Client sends challenge with authorization request
  4. Client sends verifier when exchanging code for token
  5. Server verifies the challenge matches the verifier

Security Considerations

  • Always use S256 challenge method when possible
  • Store code_verifier securely (never in URL parameters)
  • Use cryptographically secure random generators
  • PKCE is especially important for mobile and SPA applications

What Happens Next?

When you click the authorization URL, the user will be redirected to PingOne where they'll:

  1. See your application requesting permission
  2. Log in with their credentials
  3. Review and approve the requested scopes
  4. Be redirected back to your application with an authorization code
