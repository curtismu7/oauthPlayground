Authorization Code
1. Configure
Client Credentials
Enter your OAuth client details that identify your app to PingOne.

The client_id uniquely identifies your application. The client_secret proves your app is legitimate. Together, they authenticate your app at the token endpoint.

RFC 6749 Section 2.3: Client Authentication
2. PKCE
Code Verifier & Challenge
Generate a cryptographic challenge to prevent authorization code theft.

PKCE adds a layer of security by binding the authorization code to your specific app instance. If someone intercepts the code, they can't use it without the verifier.

RFC 7636: Proof Key for Public Clients (PKCE)
3. Authorize
Redirect to PingOne
Send the user to PingOne to log in and grant your app permission.

The user leaves your app and goes to PingOne to authenticate. PingOne handles login, MFA, consent — your app never sees the password. This is why it's secure: credential handling stays centralized at the identity provider.

RFC 6749 Section 4.1: Authorization Code Grant
4. Exchange
Code → Token
Trade the authorization code for access and ID tokens.

After the user approves, PingOne redirects back to your app with a code. Your backend exchanges this code (plus client_secret) for tokens. This exchange happens server-to-server, away from the browser.

RFC 6749 Section 4.1.3: Access Token Request
5. Use
Call APIs
Use the access token to call PingOne APIs on behalf of the user.

The access token proves you have permission to act as the user. Include it in the Authorization header when calling APIs. Tokens expire (typically 1 hour), so refresh when needed.

RFC 6750: OAuth 2.0 Bearer Token Usage
Role-Specific Guide
Endpoint
/authorize, /token, /userinfo
Required Scopes
openid, profile, email
Sample Code
fetch('/api/authorize', {client_id, redirect_uri, code_challenge})
Attack Vectors
Authorization code interception, client secret leakage, CSRF, redirect poisoning
PKCE Strength
SHA256 hashing, 43-128 char verifier. Mitigates code theft if browser history compromised.
Threat Model
State parameter prevents CSRF. HTTPS mandatory. Refresh tokens should rotate.
PingOne Setup
Register app in PingOne, get client_id, generate client_secret, set redirect_uri
Configuration
Environment ID, region, worker credentials for backend token exchange
Error Handling
Handle invalid_grant (code expired), access_denied (user rejected)
