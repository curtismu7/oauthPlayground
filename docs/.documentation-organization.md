# Documentation Organization

All documentation files (`.md` files) should be placed in the `docs/` folder.

## Rules:
- ✅ All `.md` files go in `docs/` folder
- ✅ `README.md` stays in root (for GitHub display)
- ✅ Preserve directory structure when moving files (e.g., `src/v8/README.md` → `docs/src/v8/README.md`)
- ❌ Do NOT create `.md` files in root directory
- ❌ Do NOT create `.md` files in `src/` or other code directories

## Structure:
- `docs/` - All documentation files
- `README.md` - Main project README (root only)

When creating new documentation:
1. Place it in `docs/` folder
2. Use descriptive filenames
3. Organize by topic/feature if needed
